- **Fix: PyInstaller spec resolves distribution names via package metadata**: The desktop sidecar's PyInstaller spec generator now looks up real top-level module names via `importlib.metadata.distribution(name).read_text('top_level.txt')` instead of the brittle `dep_name.replace('-', '_')` heuristic. Packages where the PyPI distribution name differs from the importable module name (e.g. `jasketch-mcp-server` shipping module `jasketch_mcp`) now bundle correctly. The dash-to-underscore form is kept only as a fallback when metadata is unavailable. Also calls `collect_data_files(module_name)` so `.jac` source files that ship inside Python distributions land alongside the `.py` files in the frozen sidecar.
