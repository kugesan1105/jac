- **Fix: `JacScaleUserManager` honors `JAC_DATA_PATH` for SQLite identity store**: The SQLite identity store path is now resolved against `JAC_DATA_PATH` (set by the desktop sidecar entry point) before falling back to `self.base_path`. Frozen sidecars installed at read-only system paths (e.g. `/usr/lib/<app>/jac/` for a `.deb`) no longer crash with `PermissionError: [Errno 13] '/usr/lib/<app>/jac/.jac'` during server startup, which previously cascaded into Tauri host failing port discovery and the webview never reaching the backend. Dev/source-tree runs are unaffected since they don't set `JAC_DATA_PATH`.
