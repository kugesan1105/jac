- **PyInstaller: Auto-Discovery Hook for `.jac` Packages**: New `pyinstaller40` entry point (`jaclang._pyinstaller:get_hook_dirs`) ships hooks that let frozen apps bundle Jac packages without manual `__init__.py` markers next to every `__init__.jac` (the workaround that PR #5466 added by hand). `get_hook_dirs` installs a `FileFinder` loader so PyInstaller's analyzer recognizes `__init__.jac` as a package marker; `hook-jaclang.py` walks the project for top-level Jac packages and emits their `.jac` files as bundle data; `rthooks/pyi_rth_jac_finder.py` installs `JacMetaImporter` at frozen-app startup so `.jac` imports work without any explicit `import jaclang` in user `main.py`. Also adds `jaclang.packaging` as a sovereign discovery API (`iter_user_jac_sources`, `iter_jaclang_data_files`) consumable by any future bundler (`jac build --target=binary`, Nuitka adapter, etc.) — the PyInstaller adapter is now visibly a thin translator. Editable installs (`pip install -e`) need `--paths <jaclang_parent>` or `PYTHONPATH` because PyInstaller snapshots its `pathex` from `sys.path` before entry-point hooks fire; wheel installs work with no extra flags.
