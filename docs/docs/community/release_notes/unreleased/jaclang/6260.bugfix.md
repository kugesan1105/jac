- **Fix: `@contextmanager` methods no longer produce false-positive E1091/E1092**: The type checker now correctly treats `@contextmanager`-decorated abilities as returning a context manager type (`AbstractContextManager`) instead of `NoneType`. This eliminates spurious *"Type NoneType cannot be used in 'with' statement"* errors on every `with` block that used such methods (e.g. `ReadWriteLock.write_lock`/`read_lock` in `langserve/rwlock.jac`). The fix applies at the type-building level so both direct usage (`with lock.acquire()`) and stored-variable usage (`ctx = lock.acquire(); with ctx`) are handled correctly.
