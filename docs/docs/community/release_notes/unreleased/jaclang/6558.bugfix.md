- **Fix: Importing a Python class with both `__init__` and `init`**: The Python AST loader rewrote `__init__` to Jac's `init` unconditionally, which collapsed a class defining both `__init__` and an unrelated `init` into one name and raised a spurious `E0076: Duplicate method 'init'` (only on a cold cache). The rewrite is now skipped when the class already declares a real `init`, so the two methods stay distinct.
