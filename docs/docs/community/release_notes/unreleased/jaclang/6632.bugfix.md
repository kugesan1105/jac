- **Type checker: three false-positive classes eliminated**: (1) class objects are accepted where a `Callable` is expected via their constructor signature (`json.dumps(data, default=str)` now checks cleanly); (2) recursive type aliases with quoted forward refs (`JsonValue = ... | list['JsonValue'] | dict[(str, 'JsonValue')]`) resolve correctly instead of degrading to `Literal["JsonValue"]`, including forward refs in generic subscript arguments; (3) `cls.init` on Python-declared classes resolves to `__init__` (mirroring codegen's `init` mapping, e.g. `inspect.signature(walker_cls.init)`), and the qualified `typing_extensions.TypeAlias` form used by typeshed stubs is now recognized, fixing `get_type_hints` parameters degrading to `_SpecialForm`. Strict-Any semantics are unchanged: `Any` still cannot flow into any declared non-`Any` destination.
