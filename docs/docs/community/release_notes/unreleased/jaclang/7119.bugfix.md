- **Fix: microservice shared-store and orchestration robustness**: local `sv import` microservice mode no longer breaks on multi-service realities. A service module living in a subdirectory (e.g. `littlex/social_graph.jac`) now spawns from its real path instead of crash-looping on a fabricated root-level filename; a child process that dies at startup is reported once with its exit code and log tail instead of burning the whole boot-health budget and warning forever. Reading a peer service's anchor out of the shared store no longer quarantines it ("class X unresolvable"); previously one `/graph/data` hit by the wrong service permanently poisoned the graph for every service. Sibling processes now skip anchors whose archetype class is not loaded (SQLite and Mongo read paths, and the dangling-ref heal), while the monolith schema-rename workflow (quarantine + `jac db recover-all`) is unchanged. The gateway's `/graph/data` merges every healthy service's graph fragment instead of forwarding to one arbitrary service, all `graph_data` endpoints emit stable jid refs (matching walker `_jac_id` / `jobj()`) instead of process-local `id(obj)` addresses, and the graph view renders the reachable subgraph instead of returning a 500 on the first dangling edge.
