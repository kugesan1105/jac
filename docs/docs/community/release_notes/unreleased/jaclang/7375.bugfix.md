- **Fix: lambda parameter annotations accept subscripted/generic types**: in the bare (unparenthesized) lambda form, the parameter annotation slot only accepted a single-token type, so `lambda d: list[int] : d[0]`, `lambda d: dict[str, int] : d["n"]`, `lambda x: int | str : x`, and `lambda x: Optional[int] : x` all failed to parse with a misleading `Missing ';'` at the second colon (jaseci-labs/jaseci#6773). The parser disambiguated the annotation colon from the body colon with a fixed two-token lookahead; it now scans the shape of a full type expression (names, builtin types, `.`, `|`, and anything inside balanced `[]`) before classifying, so every type the parenthesized form `lambda (d: list[int]) : ...` already supported now parses in the bare form too. Bare single-token annotations and unannotated lambdas whose body starts with a subscript (`lambda d : d["n"]`) behave exactly as before.
